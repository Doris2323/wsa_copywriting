# 水球潘（Waterball Pan）寫作風格分析報告

## 一、人物背景概覽

水球潘是「水球軟體學院」的創辦人與主講師，同時是軟體技術公司的技術創業者。他自稱兼具「創業者」與「軟體工程師」兩種視角，專注於軟體工程方法論（BDD、TDD、DDD、Design Pattern）的教學與推廣。曾於 JCConf 進行技術演講，長期經營線上課程與社群。

---

## 二、整體寫作風格定位

**一句話總結：水球潘的文風是「技術佈道者」式的寫作——以充滿激情的口語化表達，將硬核軟體工程知識包裝成具煽動力的敘事。**

他不是寫「技術文件」，而是在寫「技術演說稿」。每一篇文章讀起來都像是一場演講的文字版，有開場吸引注意、有問題拋出、有論證推演、有情緒高潮、有收尾號召。

---

## 三、語氣與口吻特徵

### 3.1 對話式語氣（最顯著特徵）

水球潘寫作的最大特色是**直接對讀者說話**，彷彿面對面教學：

- 「我問你，你覺得 AI 最大的價值是什麼？」
- 「你想想，上面隨便列一下就發現，你一份純自然語言的需求下，有這麼多潛台詞啊啊啊啊」
- 「你說說 Code Review 的人真的會開心嗎？」
- 「欸水球！在讀原始碼的時候，總會覺得特別傷眼⋯⋯」
- 「你還記得是為什麼嗎？這裡再簡單說一次——」

**特點：大量使用第二人稱「你」，製造一對一教學的親密感。**

### 3.2 自信且帶權威感

他的論述帶有強烈的個人確信，敢於下斷言：

- 「這兩句話會發生在接下來的一年內。」
- 「市面上 99% Vibe Coding 的實踐都仍在錯誤的方向上」
- 「我已經想了好幾千次這句話有沒有錯，但目前為止，這句話聽起來正確無誤。」
- 「只有不精準的情境，絕對沒有似是而非的設計！」
- 「這全部都是不負責任的說法」

**特點：使用「絕對」「一定」「99%」「必定」等極端詞彙強化說服力。**

### 3.3 熱血激勵型（佈道者風格）

文末或關鍵段落經常出現鼓舞性的號召：

- 「BDD 勝利就是咱軟體工程師的勝利。」
- 「那我們就，勇敢地一起轉型吧，不要再當工程師螺絲釘了，接下來我們要當需求工程專家。」
- 「如果你和我一樣，熱愛軟體工程、堅信軟體工程方法的價值，那麼你一定要立刻與我們一起泡在研究中，從中綻放自己身為軟體工程師的靈魂」
- 「成為在 AI 時代下不可能被取代的 Coding 高手。」

**特點：用「我們」「一起」營造共同體感，同時暗示不行動的後果（被淘汰、焦慮）。**

### 3.4 坦率直白，不迴避尖銳表述

他不怕得罪讀者，敢於直說刺耳的話：

- 「不寫題目還想變強嗎？不可能！」
- 「非軟體工程師請勿報名！！」
- 「如果你不會寫規格，不懂軟體工程，只懂得一直反覆下 Prompt 與 AI 來來回回⋯⋯那麼你並不是在善用 AI，而是在讓 AI 扯你後腿！」
- 「不怕你發問，只怕你不問！」
- 「一般那種過度靠『溝通黑魔法』在做 Vibe Coding 的做法既不可靠也無法複製。」

## 四、用字與修辭特徵

### 4.1 口語化用詞

水球潘大量使用口語化、甚至網路語言風格的詞彙：

| 口語用詞 | 出處/語境 |
|---------|----------|
| 「學爆」 | 「你不只會學爆方法論」 |
| 「多爽」 | 「AI 就能幫你全自動化開發，這樣多爽」 |
| 「吼～」 | 「吼～這種 80% 正確 20% 錯在細節的程式吼」 |
| 「啊哈——」 | 段落轉折時使用 |
| 「馬後砲」 | 技術語境中用口語描述行為 |
| 「扯你後腿」 | 描述 AI 的反效果 |
| 「幹其他人幹不了的」 | 描述學員目標 |
| 「嘛」 | 「果然，還是得靠可靠的軟工方法才能可靠地 Vibe Coding 嘛！」 |
| 「啊啊啊啊」 | 強調驚嘆感 |
| 「超・級・無・敵・累」 | 用中間圓點強調誇張語氣 |

### 4.2 比喻與類比（高頻使用）

水球潘極度擅長用生活化比喻解釋技術概念：

- **花與細胞**：「如果不能用一個個的細胞組出一朵花，那為什麼我們又會想要一行一行程式碼地去理解複雜系統？」
- **種子與基因**：「掌握原始碼的基因⋯⋯就得先思考『能使大量原始碼誕生的種子』究竟是什麼。」
- **老闆管下屬**：「AI 就像是用更便宜的價格請到軟體工程師當你的下屬⋯⋯你不該微管理你的下屬」
- **射箭與靶**：「我們先用 BDD 定義精確的『靶（測試）』，再讓 AI 進行『射箭（實作）』」
- **拼拼圖**：「就像拼拼圖般，一定是不斷地把新的拼圖拼進已有的拼圖上」
- **飯店櫃檯與行李小弟**：「就像是飯店的櫃檯人員⋯⋯好比是幫你提行李上樓的小弟」
- **說語言**：「英文說溜了，你就快速地看好多篇艱深的英文文章⋯⋯模式語言說溜了，你就能快速地理解或是設計出各種複雜系統。」

### 4.3 數字量化

頻繁使用數字來營造精確感和衝擊力：

- 「寫程式的工作，只佔工程師價值的 10% 到 20%」
- 「70% → 80% → 90% → 99% → 100% 自動化」分層級遞進
- 「99% 的人仍然在錯誤的方向上」
- 「46 部精緻教學影片 x 20 道實戰題目」
- 「產能直接乘十甚至百倍」
- 「從 2-4 週縮短至 5 小時內」

### 4.4 強調手法

- **粗體**：關鍵論點必用粗體標示
- **引用區塊**：用於核心金句或名人語錄
- **驚嘆號**：非常高頻，幾乎每段都有，經常連用（「！！」「！！！」）
- **問號連發**：「可是，規格到底怎樣才算寫清楚？？？」
- **破折號**：「——」大量使用，用於插入補充說明或轉折
- **括號補充**：頻繁用括號加入英文原文或額外解釋，如「實例化需求 (Spec by example)」

## 五、文章結構特徵

### 5.1 典型文章架構

```
1. 引言金句 / 大膽宣言（吸引注意力）
2. 提出問題 / 痛點（製造共鳴）
3. 批判現狀（建立危機感）
4. 引入方法論（提出解法）
5. 具體案例 / 實作示範（建立信任）
6. 總結 + 號召行動（推動轉化）
7. 課程推廣（商業轉化）
```

### 5.2 段落節奏

- **短句開場，長句解釋**：先用一句有力的斷言抓注意力，再展開詳細論述
- **問答推進**：大量使用反問句來推動論述，「為什麼？因為⋯⋯」「你覺得呢？其實⋯⋯」
- **層層遞進**：喜歡用漸進式架構（70%→80%→90%→99%→100%），讓讀者感受到不斷升級的層次感

### 5.3 連結前後文的手法

- 「上一篇文提到了⋯⋯」
- 「你還記得是為什麼嗎？這裡再簡單說一次——」
- 「以此類推⋯⋯」
- 「下一個單元，就要教你⋯⋯」

## 六、情緒操作與說服策略

### 6.1 FOMO（錯失恐懼）

- 「OpenAI 的研究員都這麼說了，你還在等什麼？」
- 「不學習 AI 就等著被淘汰吧！」
- 「預購 50% 現折優惠僅限預購期間⋯⋯價格將會分階段調漲」
- 「預購期結束後購買的學員⋯⋯無法參與直播互動，也無法觀看先前直播之回放」

### 6.2 認同建構

- 「如果你和我一樣，熱愛軟體工程⋯⋯」→ 篩選出目標族群
- 「BDD 勝利就是咱軟體工程師的勝利」→ 建立「我們 vs 他們」
- 「你受夠這種職涯焦慮了」→ 代入讀者情緒

### 6.3 權威引用

- 引用 DevOps 教父 Dave Farley 的話作為背書
- 引用 Christopher Alexander 的設計理論
- 提及自己在 JCConf 的演講經歷
- 「水球不是整天坐在閣樓研發知識的人，我是經營一間軟體技術公司的技術創業者」

### 6.4 痛點-解法對比

先描繪「沒有方法」的痛苦場景，再展示「有方法」的美好結果：
- 「不再當 AI 的保姆」（痛點）→「正式解放勞動力」（解法）
- 「反反覆覆與 AI 來回下 Prompt」（痛點）→「一次到位地把功能上線」（解法）

## 七、獨特的標誌性表達

以下是水球潘最具辨識度的表達方式，可視為其「文字指紋」：

1. **「水球我⋯⋯」**：第三人稱+第一人稱混用，如「水球我的想法真的很簡單」
2. **「快狠準」**：高頻使用的三字口訣，如「快狠準地實作每個功能」
3. **「絕對」「一定」「必定」**：極度頻繁的斷言用語
4. **反問連發**：連續 2-3 個反問句推動論點
5. **「⋯⋯嘛！」「⋯⋯囉！」**：口語化語尾助詞
6. **括號夾註英文**：幾乎每個技術名詞都附上英文原文
7. **「是的，⋯⋯」**：用於確認前文觀點後展開新論述
8. **「那麼⋯⋯」「因此⋯⋯」「換句話說⋯⋯」**：高頻邏輯連接詞
9. **「你想像到了嗎？」「你有沒有意識到了呢？」**：邀請讀者參與思考

## 九、適用場景與模仿要點

若要模仿水球潘的寫作風格，需掌握以下核心要素：

### 必備元素
1. **直接對讀者說「你」**——每段都要有
2. **反問句開場**——用問題拉進讀者
3. **大膽斷言**——「99% 的人都⋯⋯」「絕對⋯⋯」
4. **生活化比喻**——每個技術概念配一個日常比喻
5. **口語助詞**——「嘛」「囉」「吼」「啊」
6. **數字量化**——用具體數字增加說服力
7. **中英夾雜**——技術名詞附括號英文
8. **驚嘆號**——比一般寫作多 3-5 倍
9. **先破後立**——先批判錯誤做法，再提出正確方法

### 避免事項
1. 不要寫得像學術論文——太正式會失去特色
2. 不要迴避爭議性觀點——水球潘敢於表態
3. 不要只講理論——必須配具體案例
4. 不要冗長——短句與長句交替，保持節奏感

## 十、經典句型模板

以下整理水球潘最常用的句型結構，可作為文案撰寫參考：

```
【斷言型】
「在 AI 時代下最受重用的工程師，是最會 _____ 的工程師。」

【反問型】
「你覺得 _____ 最大的價值是什麼？是 _____ 嗎？很多人會回覆：_____ 。」

【批判型】
「市面上 99% 的 _____ 都仍在錯誤的方向上，太多 _____ 卻 _____ 的做法。」

【對比型】
「不再 _____（痛點），正式 _____（好處），讓 _____ 去做其他更有價值的事情。」

【號召型】
「如果你和我一樣，_____ ，那麼你一定要 _____ 。」

【遞進型】
「如果你要追求更高的 _____ ，那就要繼續追求 _____ 。」

【自問自答型】
「為什麼？其實——原因說複雜也不算複雜，有 N 者：1. _____ 2. _____ 3. _____」

【金句型】
「_____ 的勝利就是咱 _____ 的勝利。」
```
